# Changelog — V1.x.x

所有 **V1.x.x** 系列版本的重要变更记录于此。格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

---

## [1.0.5] - 2026-06-15

### Fixed

- **桌面端后端 `python311.dll` 找不到问题**：1.0.4 的修复方向错误。`resources` 映射 `"bin/NoteFlowBackend/_internal":"bin/NoteFlowBackend/_internal"` 导致 `_internal` 被安装到 `bin/NoteFlowBackend/_internal/`，但 Tauri 2 NSIS 打包 sidecar 时将 exe 放在安装根目录，`_internal` 与 exe 不在同一目录。修正为 `"bin/NoteFlowBackend/_internal":"_internal"`，使 `_internal` 与 `NoteFlowBackend.exe` 同处根目录，PyInstaller 运行时能正确加载 Python 动态库

## [1.0.4] - 2026-06-15

### Fixed

- **桌面端打包 `_internal` 资源路径错误**：`tauri.conf.json` 中 resources 映射 `bin/NoteFlowBackend/_internal` → `_internal` 修正为 → `bin/NoteFlowBackend/_internal`，修复安装后后端 sidecar 找不到 Python 运行时依赖导致启动失败的问题
- **删除已废弃的 macOS 打包配置**：`Cargo.toml` 移除 `[package.metadata.tauri.bundle.macOS]` 块

---

## [1.0.3] - 2026-06-15

### Fixed

- **桌面版启动失败**：修复 PyInstaller 打包后后端无法启动的致命问题
  - 根本原因：PyInstaller 打包环境下 sys.stdout 为 None，uvicorn 尝试调用 sys.stdout.isatty() 配置日志格式化器时触发 AttributeError: 'NoneType' object has no attribute 'isatty'，导致后端启动失败
  - 修复方案：`backend/main.py` 中 uvicorn.run() 添加 log_config=None 参数，禁用 uvicorn 默认日志配置，使用项目自带的 app/utils/logger.py 日志系统

---

## [1.0.2] - 2026-06-15

### Fixed

- **macOS 构建崩溃**：CI workflow 移除了 macOS ARM/Intel 编译矩阵，仅保留 Windows x64 构建，避免 macOS runner 环境兼容问题导致的持续失败

### Changed

- **桌面端后端不再弹出黑窗口**：PyInstaller 打包增加 `--noconsole` 参数，Windows 下后端在后台静默运行
- **桌面端构建前固定 setuptools 版本**：`pip install "setuptools<74"` 避免新版 setuptools 移除 pkg_resources 导致 ctranslate2 导入失败、后端一闪而退

### Internal

- **CI 更新日志链接**：release-desktop.yml 中的 release notes 链接从 V0.x.x.md 更新为 V1.x.x.md
- **CI 新增 Cargo 和 pip 依赖缓存**：actions/cache@v4 缓存 Rust crate 注册表和编译产物 + Python pip 包，未改依赖时复用缓存，预计节省约 20 分钟构建时间

## [1.0.1] - 2026-06-15

### Fixed

- **macOS 桌面端构建失败**：删除 tauri.conf.json 中错误的 macOS.files 配置（误将 PyInstaller _internal 目录声明为 macOS Framework，导致 Tauri 2 构建异常退出）
- **Docker 镜像推送 GHCR 失败**：ghcr.io/super-mortal/noteflow/backend 斜杠路径属于组织空间，个人用户不可用。改为扁平命名 ghcr.io/super-mortal/noteflow-backend，同步修正 docker-compose.yml 和 docker-push.yml 中所有镜像地址
