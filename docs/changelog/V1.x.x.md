# Changelog — V1.x.x

所有 **V1.x.x** 系列版本的重要变更记录于此。格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

---

## [1.0.2] - 2026-06-15

### Fixed

- **macOS 构建崩溃**：CI workflow 移除了 macOS ARM/Intel 编译矩阵，仅保留 Windows x64 构建，避免 macOS runner 环境兼容问题导致的持续失败

### Changed

- **桌面端后端不再弹出黑窗口**：PyInstaller 打包增加 `--noconsole` 参数，Windows 下后端在后台静默运行
- **桌面端构建前固定 setuptools 版本**：`pip install "setuptools<74"` 避免新版 setuptools 移除 `pkg_resources` 导致 `ctranslate2` 导入失败、后端一闪而退

### Internal

- **CI 更新日志链接**：`release-desktop.yml` 中的 release notes 链接从 `V0.x.x.md` 更新为 `V1.x.x.md`
- **CI 新增 Cargo 和 pip 依赖缓存**：`actions/cache@v4` 缓存 Rust crate 注册表和编译产物 + Python pip 包，未改依赖时复用缓存，预计节省约 20 分钟构建时间

## [1.0.1] - 2026-06-15

### Fixed

- **macOS 桌面端构建失败**：删除 `tauri.conf.json` 中错误的 `macOS.files` 配置（误将 PyInstaller `_internal` 目录声明为 macOS Framework，导致 Tauri 2 构建异常退出）
- **Docker 镜像推送 GHCR 失败**：`ghcr.io/super-mortal/noteflow/backend` 斜杠路径属于组织空间，个人用户不可用。改为扁平命名 `ghcr.io/super-mortal/noteflow-backend`，同步修正 `docker-compose.yml` 和 `docker-push.yml` 中所有镜像地址
