# Changelog — V0.x.x

所有 **V0.x.x** 系列版本的重要变更记录于此。格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [0.0.7] - 2026-06-13

### Added

- **新增删除供应商 API**：后端 `POST /delete_provider` 路由，支持删除供应商及其所有模型配置
- **右键删除供应商**：左侧列表每行绑定 `onContextMenu`，弹出右键菜单确认后删除
- **全选/取消全选 toggle**：拉取模型弹框中，全选按钮改为单一 toggle 状态，显示已选/可选数量
- **弹框减号快速删除模型**：拉取模型弹框中，已添加的模型显示 `[-]` 按钮可直接移除
- **Base URL 智能补齐开关**：右侧 API 凭证区域新增 `🔧自动补齐` 开关（默认关闭），开启后测试/调用时自动补齐 `/v1/chat/completions`，不修改用户输入的原地址

### Fixed

- **保存后页面空白修复**：`Model.tsx` 中保存成功后不再清除 `selectedId`，保持当前选中供应商显示
- **自动补齐不修改界面**：补齐逻辑改为仅在实际调用时生效（显示绿色提示文字告知实际请求地址），不再通过 `onBlur` 修改输入框内容

### Changed

- **模型供应商页面第三轮优化**：
  - 左侧每行绑定右键菜单，支持快速删除供应商
  - 右侧新增 Base URL 自动补齐开关，原始地址保持不变

## [0.0.6] - 2026-06-13

### Changed

- **模型供应商页面（/settings/model）重构**：
  - 保持左右分栏布局，左侧改为单列列表，每行一个供应商
  - 右侧配置面板改为内联显示，不再通过路由跳转编辑，保存后自动刷新列表
  - 删除左侧列表中名称下方的"已启用/已禁用"状态文字
  - 简化 App.tsx 中 Model 的嵌套路由，移除 ProviderForm 路由配置
- **模型供应商页面第二轮优化**：
  - **左侧列表优化**：宽度从 `w-72` 收窄为 `w-56`，名称字号加大为 `text-base`，每行右侧新增 Switch 开关控制启用/禁用状态
  - **右侧 UI 重新设计**：API 凭证区域与模型管理区域均使用品牌色左边框指示（`border-l-2 border-primary`），占满右栏空间，内容超出时支持滚动（`<ScrollArea>`）
  - **去除多余提示文字**：删除"自定义供应商需要兼容 OpenAI SDK 规范"等 Alert 提示
  - **拉取模型改为弹框选择**：点击"拉取列表"按钮后弹出 Dialog 居中展示可用模型列表，支持搜索过滤、逐行 [+] 添加、勾选批量添加，已添加的模型灰显"已添加"状态
  - **新增组件**：`ModelPickerDialog.tsx` — 通用的模型选择弹框组件
- **设置页侧边栏精简**：`SettingLayout.tsx` 移除多余的设置齿轮图标，只保留品牌标识 + 返回首页按钮

### Removed

- **删除 Groq 内置供应商**：从 `builtin_providers.json` 中移除 Groq 条目（Groq 转写配置已迁移到 `/settings/transcriber`

## [0.0.5] - 2026-06-13

### Changed

- **音频转写配置页面（/settings/transcriber）视觉重构**：重写 `Transcriber.tsx`
  - 顶部一句话简介："将视频中的语音转换为文字，供 AI 生成笔记"
  - 引擎选择采用两栏卡片布局（转写引擎卡片 + 模型管理卡片），各引擎按类型动态展示不同配置区域
  - 新增底部折叠"各引擎使用说明"，以四格卡片形式介绍 Faster Whisper / 必剪快手 / Groq / OpenAI 兼容
  - 对齐品牌色 UI（`bg-white` + `border-border/40` + `shadow-sm` 圆角卡片风格）
- **Groq 转写器配置迁移**：Groq 的 API Key 和模型名称改为在 `/settings/transcriber` 页面直接配置，不再依赖 `/settings/model` 中的供应商
  - Base URL 由系统预设（`https://api.groq.com/openai/v1`），固写不可修改（🔒 锁定图标）
  - 用户仅需填写 API Key，模型可选（预设 `whisper-large-v3` / `whisper-large-v3-turbo` / `distil-whisper-large-v3-en` / `whisper-1`）
  - 新增"测试连通性"按钮，调用后端 `/transcriber_test` 验证 API Key 有效性
  - 修改后端 `groq.py`：不再导入 `ProviderService`，改为从 `TranscriberConfigManager` 读取凭证
- **后端转写器配置扩展**：`TranscriberConfigRequest` / `transcriber_config_manager` 新增 `groq_api_key` / `groq_model` 字段

### Added

- **新增转写器连通性测试接口**：`POST /transcriber_test`，接收 `base_url` / `api_key` / `model`，调用 OpenAI `models.list()` 验证接口有效性

## [0.0.2] - 2026-06-13

### Changed

- **删除全局顶部导航栏（TopNavbar）**：
  - 从 `HomeLayout` 移除顶部导航栏，将导航功能整合到左侧边栏

- **主页侧边栏头部重构（SidePanel）**：
  - 采用品牌标识 + 设置按钮的水平布局
  - 左侧：NoteFlow Logo + 品牌名称
  - 右侧：设置按钮（从原 TopNavbar 右上角移至此处）
- **设置页侧边栏头部优化（SettingLayout）**：
  - 保留品牌标识和返回按钮
  - 新增设置图标作为当前页面指示（主题色高亮）
  - 右侧操作区：设置图标 + 返回首页按钮

### Fixed

- **修复平台图标尺寸错乱**：SVG 图标组件（`BiliBiliLogo`、`YoutubeLogo`、`DouyinLogo`、`KuaishouLogo`、`LocalLogo`）存在严重显示问题
  - **问题根源**：SVG 内部硬编码 `width="200" height="200"` 与外部容器 CSS `h-4 w-4`（16px）冲突，导致彩色块重叠、图标巨大化
  - **修复方案**：
    - 移除所有 SVG 的 `width`、`height`、`version`、`t`、`p-id` 等冗余属性
    - 移除 `className="icon"` 避免全局样式干扰
    - 保留 `viewBox="0 0 1024 1024"` 用于自适应缩放
    - 添加 `className="h-full w-full"` 使 SVG 正确继承容器尺寸
  - **影响文件**：`BillNote_frontend/src/components/Icons/platform.tsx`
- **优化图标容器布局**：`PlatformIconGroup` 的图标容器添加 `flex items-center justify-center`，确保 SVG 在 `h-4 w-4` 容器内居中显示

## [0.0.1] - 2026-06-12

### Added

- **更新日志归档规范**：新增 `docs/changelog/` 目录，更新日志按语义化版本主版本号（X）分组归档，同 X 系列版本写入同一 markdown 文档（如 `V0.x.x.md`），格式遵循 Keep a Changelog。
- **AI 编写指南**：`docs/changelog/CLAUDE.md` 记录完整的更新日志编写规范，含文件结构、命名规则、格式模板、变更类型和编写要点。
- **根目录指引**：`CLAUDE.md` 新增 Changelog 章节，指引 AI 前往 `docs/changelog/CLAUDE.md` 了解详细规范。

### Changed

#### UI 重构（Phase 1 — 基础框架）

- **顶部导航栏（TopNavbar）**：新增全局导航栏（毛玻璃效果），包含品牌标识、后端健康指示灯（桌面端 Tauri 显示）、设置入口；**无用户相关元素**
- **布局重构：三栏 → 两栏**：首页从三栏可拖拽布局（23%+16%+61%）重构为两栏固定布局（300px 侧边栏 + 弹性主内容区），侧边栏可折叠/展开带动画过渡（300ms ease）
- **侧边栏标签切换（SidePanel）**：侧边栏新增「新建笔记」/「生成历史」药丸式标签切换
- **顶层布局调整**：RootLayout 整体布局改为 `flex-col h-screen`；App.tsx 移除旧版浮动 `BackendHealthIndicator`
- **CSS 变量体系升级**：`index.css` 新增 `--surface-*` 表面层级、`--gradient-*` 品牌渐变、`--success/warning/error` 语义色、`--text-*` 文字层级等 CSS 变量
- **删除冗余文件**：`indexa.css`（与 `index.css` 完全重复）

#### UI 重构（Phase 2 — 核心页面）

- **NoteForm 重构**：
  - 提交按钮改为渐变 `bg-gradient-to-r from-primary to-indigo-500`，增加 Sparkles 图标
  - 平台选择从下拉 Select 改为 `PlatformIconGroup` 图标按钮组
  - 高级选项（视频理解、笔记格式、备注）折叠在 `CollapsibleSection` 中
  - **所有表单字段、校验逻辑、提交流程保持不变**
- **MarkdownViewer 重构**：
  - 集成 `ViewTabs`（Markdown / 时间轴 / AI 摘要 / 导图）
  - 加载态从 StepBar 升级为 `PipelineProgress` 流水线可视化组件
  - 空闲态展示 5 步引导图，操作栏使用 `StatusBadge`
  - **Markdown 渲染逻辑、复制/下载/锚点跳转等功能不变**
- **History & NoteHistory 重构**：`History` 简化头部（含记录数统计）；`NoteHistory` 搜索框加 Search 图标，使用 `StatusBadge` 替换内联状态文字，hover 显示删除按钮，添加相对时间显示；移除未使用的 `PinyinMatch` 依赖
- **新建组件**：`PipelineProgress`、`ViewTabs`、`PlatformIconGroup`、`CollapsibleSection`、`StatusBadge`

#### UI 重构（Phase 3 — 设置页面）

- **SettingLayout + Menu**：左侧导航 260px，品牌标识 + 返回按钮，菜单项活跃状态显示左侧蓝色指示条；移除 CSS 模块迁移到 Tailwind `cn()`
- **Model 页**：提供商改为 `grid grid-cols-2` 网格卡片布局；`ProviderCard` 从水平行变为垂直卡片（图标/开关/名称/状态）
- **ProviderForm 视觉刷新**：两栏卡片分区（基本信息 + 模型管理），Lucide 图标，所有提交/校验逻辑不变
- **Transcriber 两栏布局**：`grid grid-cols-1 lg:grid-cols-2` 左栏引擎配置 + 右栏模型管理
- **Monitor 仪表盘风格**：2×2 网格卡片展示四个组件状态

#### UI 重构（Phase 4 — 交互打磨）

- **CSS 动画**：新增 `animate-fade-in-up` 和 `animate-scale-in` 两个自定义 keyframe 动画，主页内容区渐入
- **骨架屏加载态**：新建 `Skeleton` 组件，Transcriber / Monitor / ProviderForm 加载态改为骨架屏布局
- **ChatPanel UI 统一**：颜色方案改为 `var(--primary)` 品牌色，头像圆角优化，边框样式统一

### Fixed

- **修复页面空白崩溃**：TopNavbar 中使用 `<Link>` 需要 Router 上下文，但因放置在 RootLayout 中无条件渲染，后端初始化完成前 `<Router>` 未挂载导致 `useContext` 返回 null 引发崩溃。修复方案：将 TopNavbar 从 RootLayout 移出，分别集成到 HomeLayout 和 SettingLayout 中

### Internal

- **安装依赖**：`@radix-ui/react-collapsible`
- **删除文件**：`indexa.css`、`components/Form/modelForm/components/index.module.css`、`pages/SettingPage/components/index.module.css`
- **删除文件**：`ThemeToggle` 组件、`github-markdown-dark.css` 引入、暗色模式检测逻辑和 `.dark` 样式块
- **构建验证**：所有阶段均通过 `vite build`，项目正常运行
- **功能不变性**：不涉及后端 API 变更、数据模型变更和用户系统；主流程保持不变
