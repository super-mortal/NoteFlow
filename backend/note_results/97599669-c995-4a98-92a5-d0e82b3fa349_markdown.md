# workbuddy 应用挑战赛（PPT大师）笔记

## 概述
- 参赛赛道：校园自动化赛道
- 技能名称：PPT大师（基于workbuddy的Skills）
- 目标：根据用户输入，自动生成原生可编辑的PPT（joML格式），支持二次编辑，而非纯图片。

## 核心工作流程
1. **接收用户输入**：用户提供文档或指定内容、风格等。
2. **初始化项目结构**：自动生成文件目录。
3. **读取设计规范**：每生成一页SVG前，重新读取规范文件，防止AI偏离主题。
4. **逐页生成SVG**：使用AI生成每页的SVG图形。
5. **质量门检测**：用Python脚本检测每页SVG是否合格，不合格则重新生成，直至通过。
6. **导出为PPT**：最终通过Python脚本将SVG映射为原生可编辑的joML文件，可在Office中直接编辑。

## 关键特性
- **原生可编辑**：与传统生成图片的方式不同，输出文件每个模块都可独立编辑，文字、图形均可修改。
- **质量保障**：每页生成前重新读取规范，并设质量门循环检测，避免AI“幻觉”导致内容偏离。
- **低积分消耗**：每次生成仅消耗2-5积分，成本较低。

## 交互式使用演示
- 用户在workbuddy中调用PPT大师技能，可选择“快速模式”（默认数据）或“交互模式”。
- **交互模式优势**：用户可自定义数据、配色方案（如科技蓝）、图标等，生成效果更美观。
- 示例配置：技术团队、科技发布风格、配色：科技蓝，性能数据准确。
- 生成过程：初始化解题结构 → 生成规范文档 → 逐页生成SVG → 质量检测 → 导出PPT文件。
- 最终PPT内所有内容均可二次编辑，风格一致。

## 示例对比
- 默认快速模式生成的PPT配色不够美观。
- 交互模式生成的PPT在配色、数据准确性上明显更优（如第三页、第四页风格统一，数据准确）。
