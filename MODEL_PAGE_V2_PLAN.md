# /settings/model 第二轮优化方案

## 一、需要解决的问题

1. **Groq 在已有数据库用户中未消失** — `builtin_providers.json` 已删除 Groq，但已有用户的 SQLite 数据库中仍存在，需要启动时清理
2. **左侧列表宽度优化** — 细化，增加每行的启用开关，名称加大，删除状态文字
3. **右侧 UI 重新设计** — 占满区域、品牌主色调、可滚动、去除多余提示文字
4. **拉取模型列表改为弹框模式** — 在页面中央显示 Modal，列出可用模型，用户点击 + 号添加

## 四、拉取模型 → 弹框选模型

### 触发流程

```
用户点击 [从 API 拉取]
        ↓
显示 Loading 状态
        ↓
弹框在页面中央打开：
┌──────────────────────────────────┐
│  可用模型  (共 24 个)      [关闭] │
│  ────────────────────────────── │
│  □ gpt-4o            [+] 已添加 │  ← 灰色不可点
│  □ gpt-4o-mini       [+]       │  ← 可点击
│  □ gpt-4-turbo       [+]       │
│  □ gpt-3.5-turbo     [+] 已添加 │  ← 灰色不可点
│  ...                           │
│                                 │
│  已选 0 个  [添加选中]          │
└──────────────────────────────────┘
```

两种操作模式：
1. **逐个添加** — 点击每行右侧的 [+] 按钮，直接添加该模型，添加后变灰色"已添加"
2. **批量添加** — 勾选左侧复选框，底部 [添加选中] 按钮批量操作

弹框：
- 使用项目中已有的 shadcn `Dialog` 组件
- 居中显示，背景遮罩
- 点击外部或 [关闭] 关闭
- 支持搜索过滤模型列表

## 五、Groq 数据库清理

在 `provider_dao.py` 或 `main.py` 的 startup 流程中增加：

```python
def cleanup_groq_provider():
    """删除数据库中残留的 Groq 内置供应商"""
    db = next(get_db())
    try:
        provider = db.query(Provider).filter_by(id="groq").first()
        if provider:
            db.delete(provider)
            db.commit()
            logger.info("已清理残留的 Groq 内置供应商")
    finally:
        db.close()
```

在 `lifespan` startup 的 `seed_default_providers()` 之后调用。
