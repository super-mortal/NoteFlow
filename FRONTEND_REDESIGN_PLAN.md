# NoteFlow 前端 UI 重构方案

> 基于 BiliNote（bilinote.app）设计语言与用户体验，对 NoteFlow 进行全面的前端 UI 重构。
>
> **重构目标**：现代化、一致性、高可用性、桌面/Web/扩展三端统一体验。
>
> **约束条件**：
> - ⚠️ **只做 UI 重构，所有功能保持不变**
> - ⚠️ **NoteFlow 无用户系统** — 无注册、登录、用户管理功能，所有参考设计中的用户相关元素（头像、菜单、账号设置等）均不纳入
> - ⚠️ **后端 API 接口不变** — 不修改请求/响应结构，仅调整前端展示方式
> - ⚠️ **现有交互流程不变** — 粘贴链接→选择配置→生成笔记→查看结果的流程不变

## 1. 设计语言

### 1.1 风格定位

| 维度 | 当前 NoteFlow | 目标风格（BiliNote 启发） |
|------|--------------|--------------------------|
| 视觉密度 | 中等，信息紧凑 | 呼吸感更强，间距更大 |
| 圆角 | `0.625rem`（10px） | 统一 `12px`，大组件 `16px` |
| 阴影 | 少量 | 多层柔和阴影，hover 时提升层级 |
| 图标风格 | Lucide 线性图标 + 平台自定义 SVG | 统一 Lucide 线性图标 + 平台彩色 SVG |
| 字体层级 | 不明显 | 清晰的 6 级字阶（xs/sm/base/lg/xl/2xl） |
| 卡片风格 | 纯白背景，无区分 | 浅灰卡片 (`bg-muted/30`) + 白色内容卡片 |
| 分割线 | `border-neutral-200` | 更细 (`border-border/50`) + 视觉淡化 |

### 1.2 设计原则

```
1. 内容优先 —— 笔记预览是核心，占据最大空间
2. 渐进式呈现 —— 引导用户一步步完成而不是一次性展示所有选项
3. 状态可视化 —— 每个阶段都有清晰的视觉反馈（动画 + 文字 + 进度）
4. 一致性 —— 跨页面、跨组件的视觉语言统一
```

## 2. 色彩系统

### 2.1 当前色彩（保留并优化）

当前主色 `#3c77fb`（蓝色）与 BiliNote 的主色高度一致，**保留为主色**，仅微调辅助色。

### 2.2 优化后的色彩体系

```css
:root {
  /* 主色 - 保留并加强 */
  --primary: #3c77fb;
  --primary-light: #e0eeff;        /* 背景色 */
  --primary-lighter: #f0f6ff;       /* 极浅色背景 */
  --primary-foreground: #ffffff;

  /* 品牌渐变 - 新增 */
  --gradient-primary: linear-gradient(135deg, #3c77fb 0%, #6366f1 100%);
  --gradient-hero: linear-gradient(180deg, #f0f6ff 0%, #ffffff 100%);

  /* 语义色 - 新增，仅 UI 展示用 */
  --success: #10b981;
  --warning: #f59e0b;
  --error: #ef4444;

  /* 面板/卡片背景 - 新增 */
  --surface-card: #ffffff;
  --surface-muted: #f8fafc;
  --surface-sidebar: #fafbfc;
}
```

### 2.3 暗色模式调整

暗色模式下主色保持不变，背景层级更丰富：

```css
.dark {
  --primary-light: rgba(60, 119, 251, 0.15);
  --surface-card: #1a1b1e;
  --surface-muted: #141517;
  --gradient-hero: linear-gradient(180deg, #0f1117 0%, #1a1b1e 100%);
}
```

## 3. 布局重构

### 3.1 全局布局结构（当前 → 目标）

```
当前:                          目标:
┌─────────────────────────┐   ┌─────────────────────────┐
│       RootLayout        │   │       RootLayout        │
│  ┌──┬──────┬──────────┐ │   │  ┌──┬──────────────────┐│
│  │F │Hist  │ Preview  │ │   │  │S │                  ││
│  │o │ory   │ (61%)    │ │   │  │i │   Main Content   ││
│  │r │(16%) │          │ │   │  │d │   (Flexible)     ││
│  │m │      │          │ │   │  │e │                  ││
│  │( │      │          │ │   │  │b │                  ││
│  │23%│     │          │ │   │  │a │                  ││
│  │) │      │          │ │   │  │r │                  ││
│  └──┴──────┴──────────┘ │   │  └──┴──────────────────┘│
└─────────────────────────┘   └─────────────────────────┘
```

**主要变化**：

- **三栏 → 两栏**：表单侧边栏和工作区分开，减少视觉碎片
- **左侧固定侧边栏**：约 `280px`，包含表单 + 历史切换标签
- **右侧主内容区**：预览/结果在中央，空间更大
- **侧边栏可折叠**：不影响主内容的阅读体验
- 顶部增加轻量导航栏，展示品牌标识 + 系统状态 + 设置入口
- 注意：**无用户系统**，导航栏不包含用户头像/登录/注册等相关元素

### 3.2 自适应断点

| 断点 | 宽度 | 布局行为 |
|------|------|---------|
| `sm` | < 768px | 全屏单列，侧边栏变成底部抽屉或弹窗 |
| `md` | 768-1024px | 侧边栏折叠图标 + 主内容全宽 |
| `lg` | 1024-1440px | 标准两栏布局 |
| `xl` | > 1440px | 两栏 + markdown 最大宽度限制 800px 居中 |

   - 当前平台下拉 Select 改为一行 5 个平台图标按钮，选中高亮

2. **高级选项折叠**：
   - 视频理解、备注、格式选择等非核心选项默认折叠
   - 用 `Collapsible` 组件实现，减少视觉噪音
   - 展开时加微动效（高度过渡）

3. **提交按钮强化**：
   - 使用渐变背景 `var(--gradient-primary)`
   - 悬停时阴影加深
   - 生成中显示脉冲动画 + 进度文字

4. **历史列表优化**：
   - 每个历史项显示：标题、状态标签、平台图标、相对时间
   - 活跃项高亮（左边框 + 浅蓝背景）
   - 右键菜单：重命名、删除、重新生成
   - **无用户相关的"我的笔记"/"共享"等分类**

## 6. 交互与动效

### 6.1 页面级动效

| 场景 | 动效 | 实现 |
|------|------|------|
| 侧边栏展开/收起 | 宽度过渡 300ms ease | CSS `transition` |

- **列表项 hover**：背景色渐变
- **Tab 切换**：下滑指示器过渡
- **Toast** 保持 `react-hot-toast`（或迁移到 `sonner` 后弹出位置在右下角不遮挡内容）

## 附录 A：BiliNote 设计参考要点

从 bilinote.app 提取的关键设计决策（仅借鉴 UI/UX 层面，**不涉及用户系统功能**）：

| 设计元素 | BiliNote 做法 | NoteFlow 当前问题 | 重构方案 |
|---------|-------------|-----------------|---------|
| **引导流程** | 页面中央展示从链接→笔记的完整流程动画 | 进入页面直接是工具表单 | 在空闲状态展示流水线引导图 |
| **三输出格式** | Markdown / 时间轴 / AI 摘要 标签切换 | 只有 Markdown 视图 | 新增时间轴和摘要视图 UI |
| **步骤可视化** | 5 步骤流水线，当前步骤脉冲 | StepBar 用表格文字 | PipelineProgress 可视化流水线 |
| **平台展示** | 彩色图标网格展示支持的平台 | 下拉选择单一平台 | 图标按钮组快速切换 |
| **卡片设计** | 柔和阴影、大圆角、呼吸感间距 | 边框为主的扁平风格 | 增加卡片层级和阴影 |
| **品牌色使用** | 蓝色贯穿：按钮、链接、强调元素 | 已有 #3c77fb 蓝色 | 保持并增加渐变使用 |

**NoteFlow 与 BiliNote 的功能差异**（重构中需注意避免引入 BiliNote 有而 NoteFlow 没有的功能）：

| BiliNote 有 | NoteFlow 无 | 处理方式 |
|-------------|------------|---------|
| 用户注册/登录 | — | 不引入任何用户认证 |
| 套餐/积分系统 | — | 不展示积分/套餐/付费墙 |
| 云端笔记存储 | 本地存储 | 保持 IndexedDB/localStorage |
| 共享/分享功能 | — | 不添加社交分享 |
| API Key 云端同步 | 本地配置 | 保持本地配置 |
